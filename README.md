# Byte Pair Encoding (BPE) Implementation

This project implements Byte Pair Encoding (BPE) from scratch for Hindi text data, demonstrating how tokenization algorithms work at a fundamental level.

## Overview

Byte Pair Encoding is a data compression technique that iteratively merges the most frequent pairs of bytes (or tokens) in a text corpus. It's widely used in modern NLP systems like GPT, BERT, and other transformer models for tokenization.

## Dataset

- **Source**: Hindi Wikipedia dataset (`zicsx/Wikipedia-Hindi`)
- **Sample Size**: 1000 articles from the training set
- **Text Processing**: Applied Devanagari-specific regex pattern to extract Hindi words
- **Original Text Length**: 7,199,749 characters (before regex)
- **Processed Text Length**: 7,017,580 characters (after regex filtering)
- **Number of Devanagari Words**: 248,156 words

## Implementation Steps

### 1. Data Loading and Preprocessing

1. Load the Hindi Wikipedia dataset using Hugging Face `datasets` library
2. Extract the first 1000 articles from the training set
3. Apply Devanagari regex pattern to filter and extract Hindi words:
   ```python
   devanagari_word = re.compile(
       r'(?:[\u0904-\u0939\u0958-\u0961\u0966-\u096F]'
       r'(?:[\u093C\u094D]?[\u0904-\u0939\u0958-\u0961])*'
       r'[\u0900-\u0903\u093A-\u094F\u0951-\u0957\u0962-\u0963]*'
       r')+'
   )
   ```

### 2. Character-Level Analysis

- Extract all unique characters from the text
- Initial vocabulary size: **129 unique characters**
- Create character-to-integer mappings (stoi) and integer-to-character mappings (itos)
- Encode text into integer sequences

### 3. UTF-8 Byte Encoding

- Convert text to UTF-8 bytes
- Original UTF-8 token length: **18,364,890 bytes**
- This represents the raw byte representation of the Hindi text

### 4. Byte Pair Encoding Algorithm

The BPE algorithm works as follows:

1. **Initialize**: Start with 256 base tokens (all possible byte values 0-255)

2. **Iterative Merging Process**:
   - Count all consecutive byte pairs in the corpus
   - Find the most frequent pair
   - Merge this pair into a new token
   - Replace all occurrences of this pair with the new token
   - Repeat until desired vocabulary size is reached

3. **Key Functions**:
   - `get_stats(ids)`: Counts frequency of all consecutive pairs
   - `merge(ids, pair, idx)`: Replaces all occurrences of a pair with a new token ID

### 5. Vocabulary Building

- **Target Vocabulary Size**: 5,000 tokens
- **Number of Merges**: 4,744 (5,000 - 256 base tokens)
- Each merge operation creates a new token by combining two existing tokens

### 6. Encoding and Decoding

- **Encoding**: Convert text string → list of token IDs
  - Start with UTF-8 byte encoding
  - Apply merges in order of creation (greedy matching)
  
- **Decoding**: Convert list of token IDs → text string
  - Build vocabulary dictionary mapping token IDs to byte sequences
  - Concatenate byte sequences and decode to UTF-8 text
  - Verified round-trip encoding/decoding preserves original text

## Results

### Final Configuration

- **Final Vocabulary Size**: **5,000 tokens**
- **Number of Merges**: **4,744 merges**
- **Original Token Length**: **18,364,890 bytes** (UTF-8 encoded)
- **Compressed Token Length**: **1,955,364 tokens**
- **Compression Ratio**: **9.39X**

### Compression Analysis

The BPE algorithm achieved a **9.39X compression ratio**, meaning:
- The original text required 18.36 million bytes when encoded as UTF-8
- After BPE tokenization, the same text is represented using only 1.96 million tokens
- This represents a significant reduction in the number of tokens needed to represent the text

### Example Merges

The algorithm learned meaningful patterns, starting with:
- Most frequent pair: `(224, 164)` - common Devanagari character byte sequence
- Subsequent merges built upon previous ones, creating multi-byte tokens
- Later merges combined previously merged tokens, creating even longer sequences

### Verification

- Successfully encoded and decoded sample text
- Round-trip encoding/decoding verified: `decode(encode(text)) == text` ✓

## Key Insights

1. **Language-Specific Patterns**: The algorithm naturally learned Devanagari-specific byte patterns, with the most frequent pairs corresponding to common Hindi character sequences.

2. **Hierarchical Merging**: Later merges combine previously merged tokens, creating a hierarchical structure that captures common word patterns and subwords.

3. **Compression Efficiency**: The 9.39X compression ratio demonstrates that BPE effectively identifies and reuses common patterns in the Hindi text corpus.

4. **Scalability**: The algorithm can be applied to any text corpus, with the vocabulary size being a hyperparameter that can be tuned based on the corpus size and desired compression.
